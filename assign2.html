Description

The learning outcomes of this assignment are:

  1. Knowledge working with the Ant build tool and the command-line and a text editor for developing Java applications.
  2. Experience working with Java Swing user-interfaces and their components; including simple controls (Button, Textfields) and composite ones requiring an underlying data model (Tree). Handling events generated by Swing components.
  3. Experience creating Java programs having multiple classes
  4. Experience using CLASSPATH to access pre-compiled auxilary Java classes contained in jar archives.
  5. Experience working with Serialization using Json.

The pre-requisites for this assignment are:

  1. Units 1-4 of the Ser321 course content. This includes Linux, Java, Ant, UI Swing, and Json Serialization
  2. The Development Environment for this assignment includes VirtualBox Linux with the packages ant, jdk 13, and org.json package. The org.json package is provided in the JSON course examples as a jar library of class files.

In this assignment, create a non-distributed Java Music Browser application having the ability to search for albums using Last.fm Web Services Your Solution should utilize the instructor provided view (Swing GUI), which is provided in compiled form. The Music Browser displays a collection of Albums and their Tracks in a Java Swing Tree. Base your solution on the sample project (available through the Canvas Site for the class) in assign2WebDist.jar. You are free to use the files contained in that example in constructing your solution.

You are to create a Album and Track classes. The information stored in these classes is based on the information available through the Last.fm Web Services. Your classes should be able to de-serialize the json result from a successful Last.fm Album.getInfo search, such as:

http://ws.audioscrobbler.com/2.0/?method=album.getinfo&artist=Cher&album=Believe&api_key=your-api-key&format=json

Register with the site to receive an api-key for making your requests. Your classes objects for Album and Track should be able to serialize to and deserialize from from json, thereby providing the functionality to save and restore the library to/from a json text file. The user interface provides the ability for browsing Album and Track that are in the collection, as well as results of successful searches. The user interface provides controls, buttons, and menu items for browsing and managing music. Your Music Browser should provide the functionality to add and remove albums and tracks. To view information about albums and tracks (by responding to tree node selection).

The browser's constructor should populate the library by causing the json file to be de-serialized and initializing the music library tree. Create MusicLibrary class having storage for the collection of Album information, and provides methods, such as:

  • getAlbumNames - a method to get the titles of albums stored in the library. It could return an array of string titles.
  • getAlbum - a method to get an Album object based on its title. You can assume that all albums in the library will have a unique title.
  • addAlbum - a method to add an album to the library. Used for example when the user wants to add the result of a Last.fm search to the library.
  • removeAlbum - a method to remove an album from the library.
  • saveLibraryToFile - a method to serialize the library to a json file.
  • restoreLibraryFromFile - a method to (re)initialize the library by de-serializing from a json file.

The Album class should have at least the following fields:

  • Album Name - the title of the album.
  • Artist - the artist or group name.
  • Tracks - a collection of Track objects representing the music tracks in the album.
  • Image - a string URL to the Last.fm provided image (use the extralarge image url, which is 300x300 pixels).
  • Genre - an optional collection of genre names. Some Last.fm searches provide tag names in the search results with provide genre information.
  • Run Time - Total the run time for each track in the album to get the album's total run time. Run time should be in the form: hh:mm:ss.
  • Summary - an optional summary description of the album. Some Last.fm searches provide album summary information in the search results.
You should also provide methods in your Album class for getting, adding, and removing Track objects associated with the Album.

The Track class should have at least the following fields:

  • Track Name - the title of the track.
  • Artist - the artist or group name.
  • Rank Order - an integer indicating the track number of this track. You should number all tracks in an album ignoring (don't reset to 1) multiple disks.
  • Duration - a string indicating the track's run time in the form: mm:ss
  • File Name - a string indicating the track's mp3 file name, such as ByeByeLove.mp3. This field is optional and only needed if you are going to support associating mp3 files with your library together with playback.

The user interface for client is provided in the example assign2WebDist.jar, which you will access through the Canvas site for the class. It includes an Ant build file that will compile and execute a sample client. It also includes javadocs for the instructor provided MediaLibraryGui class. The sample uses the JavaFX framework as provided by Gluon JavaFX. The framework is particular to host system and java version. The sample will run on either macos or Debian Linux. The project contains a readme.txt file which details how to get the example to run. The sample also includes serialization to/from Json in the MediaDescription and MediaLibraryImpl classes.

Another example project that demonstrates serialziation to/from Json for both C++ and Java is provided in the example groupJson.jar.

In summary, create a stand-alone Music Browser application, using Java. Your application should initialize a music library from a Json file, and have the ability to select, remove, and add music. Future assignments will build on this solution by providing a Media Library Server and connecting multiple clients to the server.

Functionality. Here is an explanation of the functionality that your application should provide, together with a description of how users should interact with your GUI control logic.

Your application can assume that all users will only use the GUI as it is designed to be used. That is, you do not need to validate user inputs. Further, you can assume that your users will only request valid searches of the Last.fm services.

The fields on the right side of the MediaLibraryGui are used to display and enter information about Albums and Tracks. File, Album, Track menu items are used to manage the library. Your code should implement all menu item selections (except play is not required for full credit). Your controller should also implement Search button clicks. The results of a search should be de-serialized to create appropriate Album and Track objects and the album should be displayed as the first album in the Tree. If the user wants to add the album to the library he/she will select the album node in the tree and then select the Album-Add menu item. Tree selections of Album or Track nodes causes the corresponding information to be displayed on the right side of the UI.

All source code you generate should be placed in the package ser321.assign2.your-asuriteid.client.

Structure of your Solution Directory

Submit your solution on Canvas as a Java archive file (jar) of the project directory: Assign2. Grading your project will be done by extracting the jar and executing the project using Ant from the command-line on Virtual Box Debian as defined as the Course Development Environment. You need not include JavaFX archive with your solution. You also do not need to include your Last.fm key with your solution, but your solution should accept the key as a command line argument. It will be graded on Virtual Box Debian Buster and these archives will be provided to your solution directory as they exist in the assign2WebDist.jar sample. The project directory should include only the following files, subdirectories and classes:

  • The Ant directive file build.xml. The build should contain targets for targets, clean, prepare, build, and execute. The execute target compiles and executes your Java Music Browser with appropriate arguments. Other examples are covered in class to provide Json serialization.
  • Subdirectory: lib contains json.jar and assign2Gui.jar
  • Subdirectory: src/client for your solution's source files.
  • Subdirectory classes used to store the compiled Java class files. The classes folder may be generated by Ant.
  • A file music.json which contains a Json initialization file for your library.
